# Budget-app（仮）
転職活動中の予算管理アプリ

## 1. サービス概要
転職活動中で収入がゼロになった不安の中、焦って転職先を間違えたくないと思っている人が、貯蓄・退職金・将来の支出を踏まえた「今使える金額」を瞬時に把握し、お金の不安を排除しつつプライベートも充実させながら、納得のいく転職活動に集中できるようになるアプリ。
転職活動中で収入がゼロになった人が抱える「今いくら使えるかわからない」という不安に対して、将来の支出を差し引いた今使える金額を自動算出することで、支出に対する不安が解消され、自信を持ってお金の使い道を判断できるようになる。


## 2. このアイデアはどこから生まれたか
### 2-1. きっかけとなった体験・感情
転職活動を始めるにあたって仕事がなくなり、収入がゼロになった。貯蓄や退職金から将来のイベント（引っ越し・車のローンなど）に使うお金を引いて、今使える金額がいくらなのかを知りたかった。
収入がゼロになったにも関わらず仕事がある時と同程度の生活を送ってしまうと、将来必要なお金がなくなってしまうことへの不安を抱いた。今どれくらい使えるかを把握することで、生活にメリハリをつけられると感じた。

### 2-2. なぜそれが「気になった」のか
普段から今を楽しみつつ将来の不安も取り除きたいという考え方を持っており、心配性な面がある。転職活動中に収入がゼロになり、お金の不安を強く感じた経験がこのアイデアのきっかけとなった。
自分は「使う場所とそうでない場所のメリハリをつけたい」という価値観を大切にしており、使える金額を把握することで今を楽しみながらも将来への備えができると考えている。だからこそ「今いくら使えるか」がわからないことが気になった。



## 3. 課題の整理
### 3-1. 表に見えている困りごと
収入がゼロになった時に、将来のことも考えながら現在の貯蓄から今どのくらいの使えるのかがわからずストレスを感じた。家計簿アプリをつけている人は日頃から使える金額を算出できるかもしれないが、普段つけていない人はこの問題が発生しやすい。

### 3-2. 本当に解決したい課題は何か
- なぜ①：転職期間中にお金を使いすぎると、引っ越しなどの将来的なイベントに対応できないから
- なぜ②：今お金が全く使えないと考えるだけでストレスが生まれ、人付き合いも悪くなるから
- なぜ③：人との関わりがなくなると、かえって精神的ストレスにつながるから

→ 本当に解決したい課題：
収入がゼロの時期に「今いくら使えるか」がわからないことで、使いすぎへの不安と使えないという思い込みの両方が生まれ、精神的なストレスにつながっている。

## 4. 想定ユーザーについて
### 4-1. 想定しているユーザー
年齢設定
- 年齢：20〜30代
- 理由：Dodaの調査によると転職した時期が最も多いのが20代後半で、次いで30代前半であることから他の年代より利用してもらえる可能性が高い。また引っ越し・結婚・車などライフイベントが重なりやすい時期でもあり、収入がゼロになった時のお金の不安を感じやすい世代であるため。

生活・状況
- すでに退職済みで収入がゼロだが退職金はある
- 実家暮らしで固定費は低い
- 引っ越し予定があり将来の支出が見えている
- 転職活動期間が数ヶ月以上になりそうで次の会社がまだ決まっていない
- 心配性だが細かい管理は苦手でざっくりとした把握をしたい

### 4-2. 自分とユーザーの距離
→ 自分自身
現在転職活動中で7月から収入がゼロになる。まさに自分自身が「今いくら使えるのか」を把握したいと思っており、このアプリは自分が使いたくて作るサービスである。

### 4-3. ユーザーがサービスを導入して利用するまでのイメージ
どんな場面・困りごとがきっかけで、このサービスを知る・思い出すか
- 転職期間中に、すでに生じ得る将来的なイベントに備えて「今どれくらいの金額使えるのだろう」と心配になった瞬間にこのサービスを思い出す。

最初の一歩（登録・導入）で、ユーザーは何をする必要があるか
- 現在の貯蓄状況と将来的に必要となる金額を入力する。それだけで今使える金額が自動的に算出される。

利用を続ける理由は何か？
- このアプリは毎日使う家計簿ではなく「大きな支出をする前に確認するアプリ」である。
心配性で将来のことが気になりながらも今を楽しみたいと思っている人が飲み会・引っ越し費用・欲しいものの購入など大きめの支出をする前にアプリを開いて
「これくらいなら使っても大丈夫」と確認することで罪悪感なく今を楽しめるようになる。
転職活動中は大きな支出に対する迷いが何度も生じるため、その不安を払拭するためにその都度アプリを開くので利用を続ける理由となる。


## 5. 既存サービス・競合調査
### 5-1. 似たサービスの調査
- サービス名：マネーフォワード ME
- URL：https://moneyforward.com/
- どんなことができるか：銀行口座・クレジットカードと自動連携して収支を自動記録・グラフ化する

### 5-2. それでも自分が作りたい理由
マネーフォワードMEなどの既存アプリは今日使った金額を入力してどのくらい支出が多かったか確認することはできる。しかし将来的な支出（引っ越し・車のローンなど）を踏まえた上で「今実際に使える金額」を自動で算出してお金に対する不安を解消することができない。
既存アプリは「過去・現在の収支を記録・管理するツール」であり、「将来のイベントへの支出を差し引いた今使える金額を知り、安心感を得たい」という自分のニーズとはそもそも目的が異なっていた。
また、スプレッドシートや電卓でも同様の計算はできるが、毎回自分で計算式を組む手間がかかり、外出先で「これ買っていいか」と迷った瞬間にすぐ確認できない。

このアプリは最初に貯蓄・退職金・将来の支出を登録するだけで今使える金額がトップ画面に常に表示され、不安な瞬間にスマホを開くだけですぐ判断できる点が既存手段にはない価値である。

### 5-3. 差別化を一文で決める
私のアプリの差別化ポイント：転職活動中に「これを買っても大丈夫か」と不安になった瞬間に、ざっくりとした概算で安心感を与えるサービス

## 6. このアプリで実現すること
### MVPで検証したい仮説
転職活動中の人は将来の支出を考慮した概算の可処分額が把握できるだけで支出に対する不安が解消され自信を持って判断できるようになる

具体的には以下の状態になれば検証できたと判断する。

- 友人とのご飯や欲しいものを購入する時にアプリで今使える金額を確認してから判断できるようになった
- お金を使うことへの罪悪感がなくなり「これくらいなら使っても大丈夫」と自信を持って判断できるようになった

### MVPで作る機能
- ログイン機能
- 現在の貯蓄・退職金の入力機能
- 転職期間中に発生しうる支出登録機能（引っ越し・車のローンなど）
- 今使える金額の算出・表示機能

### 算出ロジック
今使える金額 =収入 - 将来の支出合計

収入側（プラス）
- 現在の貯蓄
- 退職金
- その他臨時で入ってくるもの

将来の支出側（マイナス）
- 引っ越し費用
- ローン（車など）
- 実家への家賃
- その他決まっているイベント
　（結婚式の祝儀など）

### 本リリースで作る機能
- 日ごと・月ごと・数ヶ月ごとに使える金額を算出する機能
- 使いすぎ警告の通知機能
- レシートの読み込み機能
- 入力リマインド通知機能
- 日々の支出記録・編集機能

## 7. このアプリの懸念点とその対策
### 懸念点①：継続的な入力
- 何が問題になりそうか：ユーザーが継続して支出を入力し続けてくれるか
- なぜそう思うか：家計簿を普段からつけている人はお金の管理を習慣化できるが、普段つけていない人にとって継続的な入力は難しい
- そのために考えている対策：概算での計算をするため、日々の入力は求めず大きな支出のみ入力を求める

### 懸念点②：将来の支出が変動した場合
- 何が問題になりそうか：予定していた引っ越し費用などが増減した時に対応できるか
- なぜそう思うか：将来の支出が変わると算出される金額も変わるため、都度更新できる仕組みが必要
- そのために考えている対策：将来の支出をいつでも編集できる機能を追加する

### 懸念点③：入力した金額の正確性
- 何が問題になりそうか：ユーザーが将来の支出を過小評価・過大評価した場合、算出金額が不正確になる
- なぜそう思うか：引っ越し費用など実際にかかる金額は人によって異なり、見積もりが難しい
- そのために考えている対策：イベントごとの平均費用を参考値としてアプリ内に表示し、ユーザーが正確な金額を入力しやすくする

### 懸念点④：セキュリティ
- 何が問題になりそうか：貯蓄額などの金融情報を扱うため情報漏洩のリスクがある
- なぜそう思うか：金融情報は特にプライバシーへの配慮が必要で、漏洩した場合のダメージが大きい
- そのために考えている対策：パスワードの暗号化・HTTPS通信による暗号化を実装する。登録情報はメールアドレスとパスワードのみに限定する

## 8. 今後の展開・発展の方向性
- ユーザーはなぜ使い続ける？: 転職期間中の不安感を数字で可視化することで解消できるため
- 続けることで何が変わる？: 漠然としたお金の不安が徐々に払拭され、転職活動に集中できるようになる
- データが溜まると何ができる？: 過去の支出データが蓄積され、より正確な予算の見積もりができるようになる（毎日の入力は求めない）

### 今考えている発展案
- 機能追加の方向性：日ごと・月ごと・数ヶ月ごとの算出機能・使いすぎ警告通知・イベントごとの平均費用の参考値表示
- UI・体験の改善：入力画面のシンプル化・今使える金額をトップ画面で一目でわかるよう表示・残り使える金額が減った時に視覚的にわかるデザイン
- 機能以外の工夫：転職活動が終わった後も新生活の予算管理として使い続けられる設計にする

## 9. 技術スタック
### 9-1. 使用予定の技術

| カテゴリ | 技術 | 用途 | 選定理由 |
|----------|------|------|----------|
| フレームワーク | Ruby on Rails | バックエンド・API作成 | MVP機能である貯蓄・支出の登録と今使える金額の算出でActiveRecordを利用することでDBとのやりとりを容易にできる |
| データベース | PostgreSQL | データの保存・管理 | RailsのデフォルトDBのため連携がスムーズ |
| 認証 | Devise | ユーザー認証 | ユーザーごとに貯蓄・支出データを管理するために必要。RailsのGemとして簡単に導入できる |
| デプロイ | Render | 本番環境への公開 | Herokuと同じような操作感でデプロイできるため初学者でも導入しやすい |


### 9-2. 技術選定の理由
- 今回チャレンジしたい点： 計算ロジックの実装（貯蓄 - イベントの支出 - 日々の支出 = 今使える金額）
- 不安な点： 将来の支出が変動した時の計算ロジックの複雑化。
